Attention:

This release contains a larger set of breaking changes. We expect to make no additional breaking changes to bridge. v2's functionality after this alpha release. You may need to create a new project in order for bridge. to function correctly after this update.

Features:

  • bridge.'s internal data now has a new, modularized format
    • This change enables theoretical support for other target platforms
  • Updated data to latest beta
  • bridge. now loads tga, jpg & jpeg image references correctly
  • bridge.'s compiler now restarts automatically whenever you add/remove a "com.mojang" folder
  • Custom components can now access the compiler mode ("dev"/"build") they are running in
  • Added auto-completions & validation for geometry files

Changes:

  • Enabled Go To Definition for entity families
  • Enabled Go To Definitions for BP animations
  • Reverted the change that made the data loader performance worse again
  • Renamed "bridge/" folder to ".bridge/"
  • Disabled image dragging
  • Updated internal file name: "files.json" > ".compilerFiles"
  • Updated file name: ".ignore-folders" > ".ignoreFolders"

Fixes:

  • Fixed "minecraft:overworld_generation_rules" schema
  • Fixed refreshing of projects
